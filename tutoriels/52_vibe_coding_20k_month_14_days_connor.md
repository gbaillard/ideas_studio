## Concept
**Payout** = app mobile de decouverte de class action lawsuits. Les utilisateurs trouvent les recours collectifs auxquels ils sont eligibles, remplissent les formulaires directement dans l'app, et recuperent de l'argent de grandes entreprises. App 100% vibe-codee avec Claude en 14 jours.

## Parcours du fondateur
1. Gamer sans background technique
2. Idee d'app en pleine nuit → note dans le telephone
3. Achete un vieux MacBook en vendant des objets
4. Premiere app : Hotspot Events (app sociale) → 6-8 mois de dev → echec → "les apps sociales sont trop dures a scaler solo"
5. Pivot vers **utilitaires et outils** (plus faciles a scaler en solo)
6. En 3 ans : ~6 apps, $1M+ en revenus cumules

## Process etape par etape — Playbook de vibe coding

### Phase 1 — Recherche et inspiration
1. Telecharger **~20 apps** dans la niche cible + apps avec un beau design
2. **Screenshoter chaque ecran** de chaque onboarding
3. Tout mettre sur **une ligne dans Figma** pour vue d'ensemble
4. Cherrypicker les meilleurs elements de chaque app
5. Redesigner avec son propre theme et esthetique

### Phase 2 — Design de l'onboarding (CRITIQUE)
- **90% des utilisateurs ne verront que l'onboarding** (si paywall)
- Passer autant de temps sur l'onboarding que sur l'app elle-meme
- **5 regles d'or de l'onboarding :**
  1. **Invoquer l'emotion** — les achats sont emotionnels, pas logiques
  2. **Montrer les incentives les plus forts** — comment l'app ameliore la vie
  3. **Personnalisation** — l'utilisateur doit sentir que c'est fait pour LUI
  4. **Charts et graphiques** — rendre l'app "scientifique" et credible
  5. **Social proof** — montrer que d'autres l'utilisent avec succes

### Phase 3 — Design des structures de donnees
- **AVANT de coder** : creer un document texte avec :
  - Explication des entites
  - Exemples JSON des structures de donnees
  - Description de chaque attribut
- Essentiel pour le vibe coding : l'IA n'a pas a deviner → code plus vite et mieux

### Phase 4 — Vibe coding du produit
1. Commencer par la **fonctionnalite core** (pas l'onboarding)
2. **Drop des screenshots dans Claude/Cursor** → l'IA comprend les images et genere le code
3. Iterer avec l'IA si le resultat n'est pas parfait
4. **Ne pas etre trop perfectionniste** — mieux vaut sortir vite qu'avoir un app parfaite
5. Outil unique : **Claude Code** pour tout le coding IA

### Phase 5 — Validation
- **Deux types d'apps :**
  - Version modifiee d'un existant → **deja valide** par le marche
  - Idee nouvelle → verifier sur TikTok/Instagram si les gens parlent du probleme (commentaires = gold)
- Avec la vitesse actuelle de dev IA, on peut **skip la waitlist** et just build it
- "Any app can make thousands of dollars" — il suffit de capturer un grain de sable dans un marche de centaines de millions

## Strategie de croissance (0 → $20K MRR en 50 jours)

### Etape 1 — Campagnes UGC / Influenceurs
- Partenariat avec un createur **deja dans la niche**
- Sa communaute a de la confiance → conversions elevees
- Contenu qui parait **naturel et non-sponsorise**
- Quand un format marche → le repliquer avec d'autres createurs

### Etape 2 — Constitution d'une bibliotheque de creatives
- Le contenu UGC organique = futures ads
- On sait deja quel contenu performe bien organiquement → le boost en paid

### Etape 3 — Paid Ads (objectif final)
- **Facebook Ads** = le canal le plus consistant pour la croissance
- L'algo FB equilibre : divertissement (garder les gens sur la plateforme) vs conversions (annonceurs contents)
- **Plus la video est entertainante, moins le CPM est cher**
- Trouver le sweet spot : video divertissante ET qui pousse le produit

## L'app Payout en detail
- **Fonctionnalites :**
  - Liste de class action lawsuits disponibles
  - Verification d'eligibilite
  - Remplissage de formulaire in-app
  - Generation de PDF pret a envoyer
  - Onglet "Wallet" (suivi des claims)
  - Onglet "Forms" (historique)
- **Paywall :** Weekly + Yearly (priorite au yearly pour maximiser LTV)

## Tips cles
- **"Build something simple with a great onboarding"** — le conseil #1 de Connor
- **1 a 3 bonnes features suffisent** — l'onboarding fait le gros du travail de conversion
- **Les apps utilitaires/outils** sont bien plus faciles a scaler solo que les apps sociales
- **Le vibe coding change tout** — ce qui prenait 6-8 mois prend maintenant 2 semaines
- **Ne pas etre perfectionniste** — sortir vite > sortir parfait
- **Preparer les data structures AVANT le vibe coding** — document texte + JSON = IA plus efficace
- **Les desirs humains fondamentaux** gagnent toujours : sante, argent, beaute → Payout tape dans "les gens veulent plus d'argent"
- **Facebook Ads = le canal king** pour les apps mobiles a long terme
- **UGC d'abord, paid ads ensuite** — build la library de creatives avec l'organique
