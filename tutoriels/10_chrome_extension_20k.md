## Process étape par étape — Framework complet

### Étape 1 — Choisir une plateforme cible
**Option A : Grande plateforme existante**
- Gmail, YouTube, Twitter, Facebook, Roblox
- Des millions d'utilisateurs → si seulement 2000 veulent ta feature = revenus suffisants

**Option B : Marketplace plus petite**
- Zoom, Salesforce, Notion, Figma
- Moins de concurrence, plus facile de se démarquer

### Étape 2 — Trouver l'idée
1. **Utiliser la plateforme soi-même** et noter ce qui manque
2. Aller dans les **communautés** des utilisateurs : subreddits, Discord, groupes Facebook, Slack
3. **Écouter** ce que les gens demandent / se plaignent
4. Construire la feature manquante

### Étape 3 — Builder et lancer vite
- Tech nécessaire : **JavaScript + HTML + CSS** — c'est tout
- Pas besoin de framework complexe au début
- Premier MVP en **2-3 jours**
- Valider rapidement : est-ce que des gens l'installent ?

### Étape 4 — Croissance organique
- **Zéro dépense publicitaire** → 100% organique
- Poster dans les communautés Reddit / Discord / groupes
- Les utilisateurs contents écrivent des articles "Top 10 extensions" naturellement
- Chaque feature demandée → implémentée → annoncée → boucle de croissance

### Étape 5 — Monétisation (attendre d'avoir une base)
1. **Phase gratuite** (9-10 mois) → construire la base d'utilisateurs
2. **Newsletter** : capturer les emails de la base → monétiser via sponsors ($quelques centaines/semaine)
3. **Freemium** : les features existantes restent gratuites, nouvelles features = payantes
4. Expérimenter les prix pour trouver le sweet spot

## Stack technique
- **Code :** VS Code, JavaScript pur (pas de framework)
- **Backend :** AWS
- **Newsletter :** Beehiiv
- **Sponsors :** Passionfruit

## Chiffres clés
- **MRR :** $20 000-30 000
- **Downloads :** ~270 000
- **Weekly active users :** ~150 000
- **Équipe :** Solo (1 personne)
- **Marketing payé :** $0

## Tips clés
- Le **timing** est crucial — être early sur une plateforme qui explose
- Construire pour **soi-même d'abord** → le meilleur test de marché
- La newsletter est un **canal de monétisation sous-estimé** pour les extensions
- Parler aux utilisateurs pour trouver les **niches métier** inexploitées (avocats, médecins, etc.)
- "Si tu sais construire un site web simple, tu sais construire une extension"
- Ne pas monétiser trop tôt → construire la confiance d'abord
